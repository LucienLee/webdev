Box Model

CSS Spec

margin, border, padding

心室肥大

margin 二三事

  • border 以外，和其他元件的距離
  • inline 元素的 margin-top、margin-bottom 會被忽略
  • block 元素的 margin-top, margin-bottom 密接時，有時只會表現較大的那一個 （collapsing margin）
  • margin-top 可能會超出父元素。
  • margin-left 和 margin-right 同為 auto 時，元素會置中。

border 與 padding

  • border 會佔有空間、推開周圍元素；outline 則不會。
  • padding 是 border 與內文的間隙。
  • 元素的背景顏色會填滿 border 內的空間
  • 背景顏色的寬 = 元素 width + 左右之 padding；高亦如是。

切換 box model

box-sizing: border-box;

width = border + padding + content

3 Positioning Schemes

  • Normal Flow
    display: 預設值, position: static; float: none;
  • floats
    float: left | right
  • absolute positioning
    position: absolute | fixed

三角關係

  • 若 display 為 none 則無視 position, float
  • 否則，若 position 為 absolute 或 fixed, 無視 float。 http://jsfiddle.net/49aV7/ 大部分的 inline element 的 display 被設為 block（有 padding 與 width, height 等）。
  • 否則，若 float 不是 none，則 display 被設成上面那樣。
  • 否則，使用 CSS 內設定的 display。

換句話說：

  • 如果用 display: none 把元素藏起來，那連 box 都沒有， position 和 float 都不用設。
  • 如果設了 position:absolute | fixed 就不用設 float 了。
  • 如果設了 float 就不用設 display 了。
  • 從 normal flow 中起飛的 element 可使用 width, height 等 CSS 規則

細說 display

  • none: 啥都沒有（因為沒有 box），一丁點都不剩。
  • inline: 當成 inline element，沒有 width, height 等
  • block: 當成 block
  • table 系列：當成 table 來排版，見< a href="http://www.w3.org/TR/CSS2/tables.html#table-display"對照。

細說 position

element 套用後，起飛變成 absolute positioning:

  • absolute 相對於第一個非 static 的 parent
  • fixed 相對於瀏覽器網頁視窗（viewport）

element 套用後，仍身處 normal flow 與世推移：

  • relative 不安份亂動
  • static 預設值

細說 position (2)

absolute, fixed, relative 用 left, top, bottom, right 調位置。

absolute positioning 的元素不會撐開 parent block。

細說 float

套用後自 normal flow 中起飛。

特色：其他 normal flow 圍著它（文繞圖）

不會撐開 parent，要 clear（只對 block 有用）

Z-index

Render 網頁就是蓋房子。

瀏覽器這樣蓋房子：

  1. normal flow 中的 block elements
  2. float elements
  3. 文字等 inline elements
  4. position != static 的 element

先蓋的會被壓在下面。

Z-Index

「stacking context」堆起 element，是地基。

z-index 越大，在 stacking context 中越上層。

每個 stacking context 上的房子都這樣蓋：

  • stacking context 之背景與邊界
  • 負 z-index 的 element
  • 地表：上一頁的那些東西。
    z-index:0 的 element 和 position!=static 為同一層。
  • 正 z-index 的 element

stacking context

  • stacking context 裡面也可以有子 stacking context。
  • position != static 且 z-index != auto 的 element
  • opacity < 1 的 eleent 也會被視為一個 stacking context。原因請參考 http://www.w3.org/TR/css3-color/#transparency。
  • stacking context 在同儕間會被視為不可分割的一層，其他 stacking context 沒辦法插進來。
